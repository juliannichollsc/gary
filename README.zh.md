## 🐾 用大白话说 —— GARY 是什么？

**GARY 是一款免费的桌面应用，帮你用少得多的手动工作找到工作。**

你只需上传一次简历。之后，GARY 就替你完成找工作中最繁琐的部分：

- 🔎 **同时在多个招聘网站上搜索**。
- ✅ **判断哪些职位真正适合你** —— 它会阅读真实的职位描述，而不只是标题。
- ✍️ **为每个职位定制你的简历**，让你用最有竞争力的版本去投递。
- 📋 **填写申请表单**，一直填到最后一步……**然后停下来。**

**"提交"按钮永远由你自己来按。** GARY 绝不会替你提交申请，也不会替你解验证码 —— 最终的决定和最后一次点击，永远属于你。它的目的是*辅助*你的求职：更少但更契合的职位、少得多的复制粘贴，而你始终掌控全局。

你用自然语言与 GARY 对话，它则在你自己的电脑上安静地在后台工作。

## ⬇️ 下载与安装（无需编程）

不是开发者？你什么都不用编译。前往 **[Releases 页面](https://github.com/juliannichollsc/gary/releases/latest)**，下载 `GARY_x.y.z_x64-setup.exe` 并双击。目前**仅支持 Windows**。

## 面向开发者

GARY 是一款 **桌面应用（Tauri + Rust）**，在一个友好的聊天界面背后接入了终端级的 AI 智能体。你输入内容；在底层，GARY 驱动你选择的 AI CLI（Gemini / Claude / OpenCode），它加载 GARY 的 skills + engines 来：在多个招聘网站上搜寻职位、通过阅读真实描述验证契合度、按角色定制你的简历，并准备好申请 —— **在提交前停下**（点击仍由人来完成）。

**模型/CLI 无关** —— 后端在聊天中选择；认证使用 CLI 自身的登录或存放在操作系统密钥链中的 API 密钥。所有与候选人相关的内容都是「数据」，因此它对任何人都适用：提交你的基础简历 → GARY 映射你的角色 → 按职位定制。

### 架构（PTY 桥接）
- **Shell：** Tauri（Rust 内核 + webview）—— 本地访问文件 + 启动自动化浏览器。
- **聊天 = 终端：** `xterm.js` + 一个 PTY（`portable-pty`）启动交互式 AI CLI；你输入的内容 → CLI stdin；CLI 输出 → 聊天。
- **大脑：** CLI 加载 `.claude/skills` + `.claude/agents` + `docs/operating-rules.md`（可移植、与用户无关）。
- **确定性引擎：** `engines/*.mjs`（网站抓取、评分、简历生成器）—— 约 0 token。LLM 是**监督者**，仅在需要判断时介入：意外错误、未知的申请问题，以及按 ATS 定制的简历。
- **候选人上下文 = NotebookLM RAG：** 你的简历 + 引导阶段的回答被整合进一个可查询的笔记本，因此不会重复提问，也不会把 PII 硬编码进代码。

### 语言
界面提供 **5 种语言** —— English · Español · Português (pt-BR) · Deutsch · 中文 —— 根据你的操作系统自动检测（默认英文），并可在「设置」中切换。名称「GARY」永不翻译。

### Skills（安装前已审核 —— 见 `.claude/skills/SKILLS.md`）
GSAP（动画）· ui-ux-pro-max（设计系统）· MCP Pencil（设计→代码）· NotebookLM RAG（`proyecto26/notebooklm-ai-plugin`）· 求职引擎（从 career-ops 移植）。

### 快速开始
```bash
corepack enable pnpm   # GARY 使用 pnpm（不是 npm）
pnpm install
pnpm tauri dev
```
需要 Rust ≥ 1.96 + Node ≥ 22.13 + pnpm ≥ 11（Windows：MSVC C++ Build Tools + WebView2）。构建任务清单见 `CLAUDE.md`，方法论见 `docs/operating-rules.md`。

> **平台：目前仅支持 Windows。** 安装程序以 Windows `.exe`（NSIS）形式发布。Linux/macOS 版本已在路线图中，但需要一次可移植性改造，暂不可用。

## 负责任且尊重的使用

GARY 的存在是为了帮助**你**更快找到*合适*的工作 —— **而不是**为了搞垮网站、发垃圾信息或用垃圾内容淹没任何人。它在设计上就刻意做到**克制而尊重**：

- **不对单个网站发动机器人群。** GARY 绝不会对同一个网站并发大量机器人 —— 它会**限制并发**（例如 Himalayas ≤ 2 个并发请求，并带停顿），正是为了**避免限流 / HTTP 429**，对每个平台保持温和。真正的并行是跨*不同*来源，而不是许多标签页猛击同一个网站。
- **质量优先于数量 —— 不发垃圾，不制造垃圾。** GARY 会阅读每个职位的**真实描述**并应用双向契合门槛，因此呈现的是**数量更少、更好、与你画像相符**的匹配，而不是狂发千篇一律的申请。它绝不批量投递，也不生成充数/垃圾内容。
- **最终由人来定夺。** GARY 会把申请**填到"提交"这一步就停下** —— 由*你*来审阅并点击提交。它绝不自动提交，也不解验证码。
- **尊重每个网站。** 使用一个**专用、隔离的自动化浏览器**（绝不用你的个人浏览器），尊重已关闭/过期的职位、每家公司一次申请、冷却时间，以及每个网站的反爬信号（Cloudflare、429）。如果某网站封锁自动化，GARY 会交回给你，而不是强行突破。
- **你的账号，你的责任。** 自动化已登录的招聘网站可能与其服务条款冲突；你是用**自己的**账号、按自己的判断来运行 GARY。防护措施能降低风险但无法消除 —— **请遵守每个平台的服务条款。**
- **正常地使用 AI。** GARY **不会**修改、封装、越狱或转售任何语言模型 —— 它只是把*你*选择的终端 CLI 当作一个普通的编程助手来运行，遵守你的 LLM 提供商的使用政策。

**简而言之：** 一个用来*辅助*真实个人求职的开源工具 —— 加速筛选与定制，保持低量而诚实，并把每一次提交都留给人来做。它是为了**便利**而生，而不是为了破坏流程或网站。
